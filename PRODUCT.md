# Product

## Register

product

## Users

Single user — a full-time personal trader. Uses the app daily across all life areas: trading sessions, health tracking, learning reviews, financial monitoring, mood journaling, and weekly self-review. This is a private command center, not a team tool. The user is the developer and the end user simultaneously.

Context of use: desktop-first, typically at a workstation during trading hours. The app runs in a browser tab, often alongside broker software and charts. Needs to stay calm and legible under cognitive load.

## Product Purpose

Alpha Trader is a Personal Operating System Dashboard with a full-time Trader Command Center at its core.

The data flow it serves: **Raw data → Dashboard → Charts → Insights → Actions → Review → Self-improvement**

Modules: Dashboard, Today, Work, Health & Habits, Learning, Trading, Money, Mood & Journal, Review, Calendar, Settings.

Success looks like: the user opens the app each morning, knows exactly where they stand (portfolio, tasks, health, review queue), makes informed decisions during the session, and closes the day with a clear record. No spreadsheet-juggling, no separate apps.

## Brand Personality

Premium · Calm · Sharp

Voice is precise and confident — like a tool that knows what it's doing and doesn't need to shout. No gamification, no celebration confetti. Data is the star; chrome is minimal. When something goes well, the number shows it.

Emotional goals: focused clarity, quiet confidence, a sense that the tool is always in control.

## Anti-references

- **Robinhood / generic fintech** — green-only profit color, cartoonish illustrations, confetti animations, gamified streaks. This is a serious tool for a serious trader; it must never feel like a mobile app for first-time investors.
- The app must not feel like a passive investment portfolio viewer — it needs to feel like a real-time command center.

## Design Principles

1. **Data first, chrome second.** Every pixel of decoration must earn its place. If removing it doesn't hurt comprehension, remove it.
2. **Calm under load.** The palette (pastel purples, mint, soft corals) is chosen to stay legible across long sessions. Never introduce visual noise that competes with the data.
3. **Precision at every scale.** Numbers, labels, and states must be exact and unambiguous. Approximate or placeholder copy is not acceptable.
4. **Consistent rhythm, not repetitive.** Cards and modules should share a visual language but vary in density and layout to match the complexity of their content.
5. **Actions are always clear.** Every interactive element must have an obvious affordance and a clear outcome. No mystery meat navigation.

## Accessibility & Inclusion

WCAG AA target. Body text must achieve ≥4.5:1 contrast against background. Large text and chart labels ≥3:1. Reduced motion preference respected — all Framer Motion animations must have static fallbacks. Thai + Latin text must remain legible at all size steps (the app uses Thai copy throughout).
